# Strata Implementation Status

Status date: 2026-05-23.

This document tracks where the repo currently stands against [roadmap.md](./roadmap.md). It should be updated whenever a roadmap phase materially changes.

## Current Snapshot

Strata currently has a Bun/TypeScript monorepo, an initial Markdown wiki skeleton, a model/tool agent loop with Pi-style parallel tool-call execution, shared model-adapter construction and model-listing for CLI/TUI/server callers, ChatGPT/OpenAI Codex auth, SQLite-backed session traces with CLI/TUI deletion and persisted per-assistant-message token usage, read/write wiki and filesystem tools, a shared repo-file search helper for file mentions, a dangerous-mode shell tool, registry profiles, file-change trace events, basic learning-state tools, AGENTS.md plus skill-aware run-context injection, a basic post-run reflection loop, manual maintenance jobs, a trace-backed Notion raw snapshot command, a first-party TUI package with Pi-style session launch flags, a shared connector runtime foundation, a Slack backfill/listen foundation, web-api chat run/cancel streaming endpoints with server-owned active runs, durable web chat run/event storage with replayable SSE reconnects, chat metadata procedures, tRPC chat file/model listing procedures, a local web app with connector setup, a live-verified `/chat` route over the shared agent loop with sidebar session navigation, image attachments, streamed-markdown rendering, click-to-copy session ids, file `@`-mention autocomplete, a persisted web model/reasoning picker, slash commands, local prompt history, compact token/context metrics, specialized learning-tool panels, and browser-verified dropped-stream reconnect/recovery behavior, and a formal external tool-pack/MCP architecture plan.

The system is not yet a self-maintaining wiki agent. The next major gap is closing the maintenance loop: Strata can now run named maintenance jobs manually and persist their traces, but it does not yet have proposal review/apply commands or cron-style recurring execution. Before returning to maintenance and connector work, the remaining deferred web-chat polish is responsive layout verification; dropped-stream reconnect and failed-run recovery have now been verified and tightened.
