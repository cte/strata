# Strata Implementation Status

Status date: 2026-05-24.

This document tracks where the repo currently stands against [roadmap.md](./roadmap.md). It should be updated whenever a roadmap phase materially changes.

## Current Snapshot

Strata currently has a Bun/TypeScript monorepo, an initial Markdown wiki skeleton, a model/tool agent loop with Pi-style parallel tool-call execution, shared model-adapter construction and model-listing for CLI/TUI/server callers, ChatGPT/OpenAI Codex auth, SQLite-backed session traces with CLI/TUI deletion and persisted per-assistant-message token usage, read/write wiki and filesystem tools with curated-first search ranking, a shared repo-file search helper for file mentions, a dangerous-mode shell tool, registry profiles, file-change trace events, basic learning-state tools, AGENTS.md plus skill-aware and curated-first run-context injection, a basic post-run reflection loop, manual maintenance jobs, trace-backed raw snapshot commands, a first-party TUI package with Pi-style session launch flags, a shared connector runtime foundation, a Slack backfill/listen foundation, a Granola connector that handles official cursor pagination plus detail transcript fetches, automated raw-to-wiki indexing for curated meeting/entity pages across Granola, Notion, and Slack raw snapshots, web-api chat run/cancel streaming endpoints with server-owned active runs, durable web chat run/event storage with replayable SSE reconnects, chat metadata procedures, tRPC chat file/model listing procedures, tRPC wiki tree/page procedures, a local web app with connector setup, a live-verified `/chat` route over the shared agent loop with sidebar session navigation, image attachments, streamed-markdown rendering, click-to-copy session ids, file `@`-mention autocomplete, a persisted web model/reasoning picker, slash commands, local prompt history, queued web follow-up messages, compact token/context metrics, specialized learning-tool panels, a `/wiki` route for browsing the Markdown wiki tree with an inline markdown viewer, browser-verified dropped-stream reconnect/recovery behavior, browser-verified responsive mobile/tablet/narrow-desktop behavior, and a formal external tool-pack/MCP architecture plan.


The system is not yet a self-maintaining wiki agent, but it now has an automated source-to-wiki path: raw Granola snapshots can become meeting pages, raw Notion pages can become project/source-backed project updates, and material raw Slack threads can become thread pages while updating people/project/decision/thread/action indexes, `wiki/index.md`, and `wiki/log.md`. Slack raw-to-wiki skips empty exports, automation/log notifications, link-only shares, routine status updates, and threads without a material ask/decision/action/incident/project signal. `wiki.search` now ranks curated wiki pages before raw snapshots even when `includeRaw` is enabled, and the `wiki.entities` maintenance job audits duplicate/over-specific project pages. The next major gaps are improving classification quality, applying consolidation for duplicate entities, promoting stronger decision/action extraction, and adding cron-style recurring execution.

## Immediate Next Milestone

The web chat depth milestone is complete. Dropped-stream reconnect/recovery and responsive layout behavior were browser-verified on 2026-05-23. Connector/raw-to-wiki work has resumed.

The generalized source-to-wiki slice is now present:

1. Granola raw backfill hardening is implemented in `packages/ingest/src/granolaConnector.ts`: official List Notes pagination, `created_after` filtering from `--since`, `page_size`, `maxPages`, and default Get Note detail fetches with `include=transcript`.
2. `packages/ingest/src/rawToWiki.ts` can scan raw Granola, Notion, and Slack Markdown snapshots. Granola maps to meeting pages, Notion maps to project/source-backed project updates, and Slack maps to thread pages.
3. `strata ingest raw index [--source all|granola|notion|slack] [--raw-path FILE] [--limit N] [--dry-run]` creates curated wiki pages plus lightweight people/project/decision/thread/action updates, updates `wiki/index.md`, appends `wiki/log.md`, records an ingest trace, dedupes Slack snapshots by channel/thread timestamp when scanning raw directories, and skips already-indexed raw sources. `strata ingest granola index` remains as a compatibility shortcut, and connector pulls can pass `--index`.
4. The local backfill currently has 20 raw Granola files indexed into 20 meeting pages. A filtered Slack backfill on 2026-05-24 scanned 725 deduped raw Slack threads, indexed 258 material thread pages, and skipped 467 low-signal threads. The local wiki currently has 13 people pages, 52 project pages, 33 decision pages, and 660 thread pages. Retrieval smoke tests through `wiki.search` find curated pages for queries like `document classifier` and `pricing decision` without searching raw transcripts first.
5. Current local raw data has no Notion snapshots. Post-apply Slack dry-run verification reports `would index: 0`, so the current raw Slack set is fully indexed or explicitly skipped by materiality rules.
6. Retrieval quality has a first guardrail: `wiki.search` returns curated pages before raw pages when raw inclusion is enabled, and run context tells agents to use raw snapshots only when curated pages are insufficient or raw evidence is specifically needed.
7. `strata maintain run wiki.entities` is available as a read-only quality audit. On the local wiki it currently reports two duplicate project-topic groups: Pricing and Roo Code.

The next milestone is applying entity consolidation and improving extraction quality: reduce duplicate project pages, promote stronger decision/action extraction, and keep CLI/TUI/web agent retrieval curated-first while preserving raw transcript links for evidence.
