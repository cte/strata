extension runtime. No extension loader, public extension API, or activity CLI view exists yet.

The system is not yet a self-maintaining wiki agent, but it now has an automated source-to-wiki path plus a retrieval substrate: raw Granola snapshots can become meeting pages, raw Notion pages can become project/source-backed project updates, and material raw Slack threads become source pages under `wiki/sources/slack/` while only promoted decisions/actions/projects/open threads enter the curated folders. Raw-to-wiki classification is now split between generic product logic and a local ingest taxonomy: project aliases, self-name ownership hints, and workspace-specific Slack materiality/ignore patterns load from `.strata/ingest/taxonomy.json`, while `packages/ingest/src/rawToWiki.ts` keeps generic source parsing and extraction rules and delegates taxonomy resolution/materiality/entity matching to smaller raw-to-wiki modules. Slack raw-to-wiki skips empty exports, automation/log notifications, link-only shares, routine status updates, taxonomy-marked transient checks, taxonomy-marked routine coordination, and threads without a material ask/decision/action/incident/project signal. Indexed and skipped raw-to-wiki item events now include structured classification reasons for matched aliases, material signals, low-signal filters, and action extraction run/candidate ids when action candidates are evaluated. Decision promotion still requires explicit decision language. Action promotion now routes owner-attributed raw-to-wiki commitments through `daily.todo` candidate persistence/publication before writing `wiki/actions/mine.md` or `wiki/actions/theirs.md`, so published rows get hidden extraction metadata and shared hard suppressions such as `search: 999`; Slack first-person commitments are attributed to a human-readable message speaker first, Slack user-id speakers no longer create first-person action items, and discourse markers such as "Ok" are no longer treated as person names for action ownership. The browser action manager keeps those two Markdown ledgers as the source of truth: `packages/core/src/wikiActions.ts` parses task-list rows, tRPC exposes `wiki.actions.*`, and `/actions` can filter, add, complete/reopen, annotate items with hidden `strata:action-context` comments, and review today's unpublished daily TODO candidates with evidence preview plus accept/reject controls. `@strata/ingest/extraction` now has the first six `daily.todo` slices: day-scoped corpus resolution, Slack/generic evidence spans, deterministic TODO candidates, fake-verifier tests, trace-backed `extraction.daily_todo.*` events, SQLite `extraction_runs`/`extraction_candidates`, idempotent source-span/candidate-hash persistence, an opt-in schema-checked model verifier behind `--verify`, explicit prompt/model metadata, dry-run/backfill commands, `strata extract daily-todos --date ... --apply` publication of confirmed high-confidence owned candidates to the Markdown action ledgers with source links and hidden extraction context metadata, shared web/API review for unpublished candidates, and raw-to-wiki action promotion through the same TODO publication path. `wiki/index.md` is now treated as a compact human navigation page, while `strata wiki search-index refresh` builds the local FTS index over curated pages, generated source pages, and raw snapshots. `wiki.search` uses that index when available and still ranks curated pages before raw snapshots when `includeRaw` is enabled. `wiki.entities` now emits structured consolidation groups and deduplicated pending wiki proposals instead of only flat findings, and `wiki.hygiene` makes that proposal-producing audit plus retrieval refresh schedulable. The first proposal-backed consolidation pass merged duplicate local project clusters into canonical pages, left superseded redirect notes with preserved source evidence, and taught the root index, search index, and entity audit to skip superseded project redirects. Proposals are now first-class enough to list/read/status-update from the CLI and web; explicit wiki-page creation, exact old-text wiki patches, ingest taxonomy schema operations, and exact project consolidation operation plans can be accepted without hand-copying Markdown. Consolidation apply remains guarded: the reviewed preview is fingerprinted, the plan is re-read and revalidated immediately before writing, unsafe or changed previews are refused, and successful applies write canonical patches, superseded redirects, backlink rewrites, search-index refreshes, and `wiki/log.md` entries through the shared proposal store. Newly created `wiki.entities` consolidation proposals now emit guarded exact append-only canonical merge patches for conservative project duplicates, falling back to manual review when the content is too large, empty, or ambiguous. Retrieval regression tests now cover consolidated curated pages across the core search index, `wiki.search` tool, and `strata wiki search` CLI path, including source/raw fallback ordering and superseded-page exclusion. The first representative day-by-day TODO backfill quality pass has run against local May 2026 wiki/raw content; the next major gap is adding model-verified review passes before broader publication.
