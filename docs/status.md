# Strata Implementation Status

Status date: 2026-05-30.

This document tracks where the repo currently stands against [roadmap.md](./roadmap.md). Update it whenever a roadmap phase materially changes.

## Current Snapshot

Strata has a Bun/TypeScript monorepo, Markdown wiki, model/tool agent loop, CLI, TUI, local web API, Vite web app, connector workflows, Routine triggers, proposals, ingest activity, native wiki retrieval (`wiki.search` plus chunk-level `wiki.retrieve` over a local SQLite index), raw-to-wiki indexing for Granola, Notion, and Slack, an initial `@strata/routines` package with `routine.run` job execution, Pi-aligned model thinking-level capability mapping, a clean-room experimental web terminal prototype, and plans for a native interactive agent UI primitive plus a later Pi-style extension runtime.


Raw-to-wiki currently creates meeting/source/project/person/decision/thread pages and updates `wiki/index.md`/`wiki/log.md`. It no longer promotes action items. Slack and other source classification still use generic parsing plus local taxonomy from `.strata/ingest/taxonomy.json`.

The previous automated action extraction path has been removed. This includes the `@strata/ingest/extraction` package export and implementation, daily TODO CLI commands, extraction SQLite schema/migration, web/API extraction review procedures, `/actions` extracted-candidate review queue, raw-to-wiki action publication, and activity display of daily TODO candidate ids. The generated action ledgers were cleared on 2026-05-29. The new direction is to build a general Routine primitive with structured input/output and use a Granola daily TODO Routine as the first action-intelligence producer. Routine definitions, runs, and artifacts have SQLite tables plus a typed store, and `routine.run` can now execute routines through the shared job runner and agent loop. Structured-output routines get a per-run `routine.output.submit` tool that validates submitted JSON against the routine `outputSchema`, persists valid Routine Artifacts, and marks required-output runs as `needs_review` when no valid artifact is produced. Routine definitions, runs, artifacts, and manual routine execution are now exposed through the CLI and web tRPC service layer.

The clean action starting point is now:

- `wiki/actions/mine.md`
- `wiki/actions/theirs.md`
- `@strata/core/wiki-actions`
- `wiki.actions.*` tRPC procedures
- `/actions` manual action management

The browser action manager remains useful for manual action items: it lists, filters, adds, completes/reopens, deletes, and annotates Markdown-backed action rows with hidden `strata:action-context` comments.

The planned interactive agent UI work is now captured in [interactive-agent-ui-plan.md](./interactive-agent-ui-plan.md). It should add a normal trace-backed `user.ask` tool, shared request/response types, `ToolContext.ui` and `AgentRunConfig.ui` wiring, TUI/web adapters, cancellation, non-interactive unavailable behavior, and future compatibility with extension `ctx.ui.select/confirm/input`.
