# Strata Implementation Status

Status date: 2026-05-25.

This document tracks where the repo currently stands against [roadmap.md](./roadmap.md). It should be updated whenever a roadmap phase materially changes.

## Current Snapshot

Strata currently has a Bun/TypeScript monorepo, an initial Markdown wiki skeleton, a model/tool agent loop with Pi-style parallel tool-call execution and shared append-only auto-compaction, shared model-adapter construction and model-listing for CLI/TUI/server callers, ChatGPT/OpenAI Codex auth, Anthropic Claude OAuth auth and Messages API adapter support, SQLite-backed session traces with CLI/TUI deletion and persisted per-assistant-message token usage, read/write wiki and filesystem tools with a local SQLite FTS wiki/raw retrieval index and curated-first ranking, a shared repo-file search helper for file mentions, a dangerous-mode shell tool, registry profiles, file-change trace events, basic learning-state tools, AGENTS.md plus skill-aware and curated-first run-context injection, a basic post-run reflection loop, manual maintenance jobs, trace-backed raw snapshot commands, a first-party TUI package with Pi-style session launch flags, a shared connector runtime foundation, a Slack backfill/listen foundation, a Granola connector that handles official cursor pagination plus detail transcript fetches, automated raw-to-wiki indexing for curated meeting/entity pages across Granola, Notion, and Slack raw snapshots, web-api chat run/cancel streaming endpoints with server-owned active runs, durable web chat run/event storage with replayable SSE reconnects, chat metadata procedures, tRPC chat file/model listing procedures, tRPC wiki tree/page procedures, a local web app with connector setup, a live-verified `/chat` route over the shared agent loop with sidebar session navigation, image attachments, streamed-markdown rendering, click-to-copy session ids, file `@`-mention autocomplete, a persisted web model/reasoning picker, slash commands, local prompt history, queued web follow-up messages, compact token/context metrics, specialized learning-tool panels, a `/wiki` route for browsing the Markdown wiki tree with an inline markdown viewer, browser-verified dropped-stream reconnect/recovery behavior, browser-verified responsive mobile/tablet/narrow-desktop behavior, a formal external tool-pack/MCP architecture plan, an initial Exa MCP search/fetch tool pack wired into CLI/web/TUI registries, and a planned Pi-style local extension runtime. No extension loader or public extension API exists yet.

The system is not yet a self-maintaining wiki agent, but it now has an automated source-to-wiki path plus a retrieval substrate: raw Granola snapshots can become meeting pages, raw Notion pages can become project/source-backed project updates, and material raw Slack threads become source pages under `wiki/sources/slack/` while only promoted decisions/actions/projects/open threads enter the curated wiki folders. Slack raw-to-wiki skips empty exports, automation/log notifications, link-only shares, routine status updates, and threads without a material ask/decision/action/incident/project signal. `wiki/index.md` is now treated as a compact human navigation page, while `strata wiki search-index refresh` builds the local FTS index over curated pages, generated source pages, and raw snapshots. `wiki.search` uses that index when available and still ranks curated wiki pages before raw snapshots when `includeRaw` is enabled. The next major gaps are improving classification quality, applying consolidation for duplicate entities, promoting stronger decision/action extraction, and adding cron-style recurring execution.

The generalized source-to-wiki slice is now present:

1. Granola raw backfill hardening is implemented in `packages/ingest/src/granolaConnector.ts`: official List Notes pagination, `created_after` filtering from `--since`, `page_size`, `maxPages`, and default Get Note detail fetches with `include=transcript`.
2. `packages/ingest/src/rawToWiki.ts` can scan raw Granola, Notion, and Slack Markdown snapshots. Granola maps to meeting pages, Notion maps to project/source-backed project updates, and Slack maps to source pages under `wiki/sources/slack/`; only durable decisions, actions, projects, people, and strongly promoted open threads enter the curated folders.
3. `strata ingest raw index [--source all|granola|notion|slack] [--raw-path FILE] [--limit N] [--dry-run]` creates curated wiki pages plus lightweight people/project/decision/thread/action updates, updates `wiki/index.md`, appends `wiki/log.md`, records an ingest trace, dedupes Slack snapshots by channel/thread timestamp when scanning raw directories, and skips already-indexed raw sources. `strata ingest granola index` remains as a compatibility shortcut, and connector pulls can pass `--index`.
4. `strata wiki compact-index` rebuilds `wiki/index.md` as a compact human navigation page and writes source index pages such as `wiki/sources/slack/index.md`. The root index no longer tries to enumerate every source-derived Slack thread.
5. `strata wiki archive-generated-slack-threads` archives older generated Slack thread pages out of `wiki/threads` into `.strata/archive/generated-slack-threads/<timestamp>/` and rewrites wiki links to the immutable raw Slack source.
6. `strata wiki search-index refresh [--source all|granola|notion|slack]` builds a local SQLite FTS index over curated wiki pages, generated source pages, and raw snapshots. `wiki.search` and `strata wiki search` use the index when available, rank curated pages first, then source pages, then raw evidence.
7. The local Slack backfill on 2026-05-25 processed 114 conversations, 89,409 messages, and 31,003 threads into immutable raw Slack snapshots. A cleanup pass archived 33,697 generated Slack-derived thread pages, rewrote 49,541 links to raw Slack evidence, reduced `wiki/threads` to 24 curated thread pages, reduced non-raw wiki pages to 979, and refreshed search to 35,657 indexed docs: 977 curated pages, 2 source index pages, and 34,678 raw snapshots.
